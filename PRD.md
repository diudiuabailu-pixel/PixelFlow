## 1. 产品概述

| 项目 | 内容 |
|------|------|
| **产品名称** | 待定（建议：PixelFlow / CreativeOS / VisualForge） |
| **一句话定位** | AI 原生的视觉创作工作台——从脚本到成片，一个画布搞定 |
| **核心价值** | 多模型聚合 + 无限画布 + 节点式工作流 = 人人都是视觉创作者 |
| **目标市场** | 中国 + 东南亚（中文生态优先） |
| **产品形态** | Web SaaS（浏览器端） |
| **商业模式** | Freemium + 按量付费 + 企业订阅 |

### 1.1 我们要解决什么问题？

**现状痛点：**

| 痛点 | 具体表现 |
|------|---------|
| **工具碎片化** | 用 Midjourney 生图、用 Kling 生视频、用剪映剪辑、用 Canva 排版——4个工具来回切换 |
| **模型选择困难** | Midjourney / DALL-E / Flux / 即梦，每个都要单独注册、单独付费、单独学习 |
| **工作流断裂** | 从脚本→分镜→图片→视频，每一步都是手动导出导入，上下文丢失 |
| **协作困难** | 创作过程无法分享复用，每次从零开始 |
| **门槛过高** | ComfyUI 功能强大但学习曲线陡峭，普通创作者用不了 |

**我们的解决方案：**

> 一个画布，所有模型，完整工作流。
> 
> 不需要懂 ComfyUI 的节点逻辑，不需要在 10 个 AI 工具之间切换。
> 打开浏览器，拖拽连线，从想法到成片。

### 1.2 产品愿景

**短期（6 个月）**：成为中文创作者首选的 AI 视觉创作平台
**中期（1 年）**：建立创作者社区，形成工作流分享生态
**长期（2 年）**：成为品牌和代理商的标配 AI 创意生产工具

## 5. 用户流程

### 5.1 核心创作流程

```
打开画布
    ↓
添加「文本输入」节点 → 写入 Prompt / 上传参考图
    ↓
添加「AI 生成」节点 → 选择模型 → 配置参数
    ↓
连线：输入 → 生成
    ↓
点击「生成」→ 等待（进度条）→ 结果出现在预览区
    ↓
满意 → 连线到「输出」节点 → 导出 / 保存
不满意 → 调整参数 / 换模型 → 重新生成
```

### 5.2 分镜创作流程

```
添加「脚本输入」节点 → 粘贴/编写脚本
    ↓
连线到「智能分镜」节点 → AI 自动拆分镜头
    ↓
生成分镜列表（每个镜头有描述 + Prompt）
    ↓
连线到「批量生成」节点 → 一键生成所有分镜图
    ↓
预览 → 微调个别镜头 → 导出分镜板
    ↓
（可选）连线到「视频生成」→ 图生视频 → 导出成片
```

### 5.3 电商场景流程

```
上传商品图 → 添加「商品抠图」节点 → 自动去背景
    ↓
添加「场景生成」节点 → 选择模板（白底/场景/生活方式）
    ↓
批量生成 N 个角度/场景
    ↓
添加「文案生成」节点 → AI 写卖点文案
    ↓
导出：商品主图 + 详情图 + 广告素材
```

## 6. 技术架构

### 6.1 技术栈

| 层级 | 技术选型 | 原因 |
|------|---------|------|
| **前端框架** | Next.js 14 (App Router) | SSR + API Routes + 生态成熟 |
| **UI 库** | React 18 + Tailwind CSS + Shadcn/UI | 快速开发 + 美观 |
| **画布引擎** | React Flow (Xyflow) | 开源节点编辑器，社区活跃，API 完善 |
| **状态管理** | Zustand | 轻量、性能好、适合画布场景 |
| **后端框架** | Next.js API Routes + tRPC | 类型安全、前后端一体 |
| **数据库** | PostgreSQL (Supabase) | 关系型 + 实时订阅 + 免运维 |
| **缓存** | Redis (Upstash) | 任务队列 + 限流 + 缓存 |
| **对象存储** | AWS S3 / Cloudflare R2 | 图片/视频资产存储 |
| **认证** | NextAuth.js | 多渠道登录（邮箱/微信/Google） |
| **支付** | Stripe + 微信支付 | 国际 + 国内双通道 |
| **部署** | Vercel (前端) + Railway (后端任务) | 快速部署 + 自动扩缩 |
| **CDN** | Cloudflare | 全球加速 + 国内加速 |
| **监控** | Sentry + PostHog | 错误追踪 + 用户行为分析 |

### 6.3 AI Gateway（核心组件）

```
AI Gateway 职责：
├── 模型路由    → 根据用户选择分发到对应 AI API
├── Key 轮换    → 多 API Key 自动轮换，故障自动切换
├── 黑名单机制  → 额度耗尽/失效 Key 自动拉黑
├── 任务队列    → 异步处理，支持长任务轮询
├── 结果缓存    → 相同参数缓存复用
├── 用量计费    → 记录每次调用的模型/Token/费用
└── 限流保护    → 按用户/全局限流
```

### 6.4 画布性能优化策略

| 策略 | 说明 |
|------|------|
| 虚拟化渲染 | 视口外节点不渲染 DOM |
| GPU 加速 | transform: translateZ(0) 硬件加速 |
| 缩略图缓存 | 生成 80px 缩略图避免大图加载 |
| 增量更新 | 只重绘变化的节点和连线 |
| Web Worker | 图像处理等重计算放到 Worker |
| 懒加载 | 媒体资源按需加载 |

| AI API 调用成本过高 | 高 | 利润压缩 | 缓存策略 + 模型路由优化 + 阶梯定价 |
| Midjourney API 不稳定 | 中 | 核心功能受影响 | 多通道备份 + 降级到其他模型 |
| 画布性能瓶颈 | 中 | 用户体验差 | 虚拟化 + Web Worker + 渐进加载 |
| 国内合规 | 高 | 无法上线 | 内容审核 + ICP 备案 + 敏感词过滤 |
| 用户增长慢 | 中 | 收入不达预期 | 社区运营 + KOL 合作 + 模板引流 |
| 竞品跟进 | 中 | 市场被抢 | 深耕中文场景 + 工作流生态壁垒 |
